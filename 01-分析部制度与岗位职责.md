# 第一部分：部门制度

## 一、部门定位
分析部是公司的数据智能中枢，负责数据分析、统计建模、数据可视化，将调研数据转化为可量化的洞察和决策依据。

## 二、工作原则
1. **数据驱动** —— 一切分析结论基于数据，避免主观臆断
2. **可验证** —— 分析过程可复现，模型逻辑透明
3. **可视化优先** —— 复杂数据通过图表直观呈现
4. **业务导向** —— 分析服务于决策，不提纯技术性结论

## 三、工作流程

### 3.1 数据分析流程
1. 接收调研部数据需求 → 明确分析目标
2. 数据采集/清洗 → 构建分析模型
3. 数据分析执行 → 产出分析报告
4. 可视化呈现 → 交付分析部模块

### 3.2 交付物类型
| 交付物 | 说明 |
|-------|------|
| 数据分析报告 | Excel/Python输出的结构化分析 |
| 可视化图表 | 柱状图、折线图、雷达图、热力图等 |
| 统计模型 | 回归分析、预测模型、聚类分析 |
| 数据看板 | 关键指标 Dashboard（可视化） |

## 四、质量标准
- 数据清洗：异常值处理、缺失值处理须记录
- 模型选择：说明选型依据和局限性
- 图表规范：统一配色、字体、标注风格
- 结论表达：用业务语言解释分析结果

### 数据自洽性检查清单
- **总量一致性**：分项数据加总与汇总数据一致，数值无矛盾
- **时间自洽**：时间序列数据前后衔接，无断点或跳跃无解释
- **比率验证**：计算比率（增长率、占比）与原始数据吻合，无四舍五入累积误差
- **跨表交叉校验**：不同表格/来源的同一指标数据比对一致
- **单位统一**：全文中数据单位一致，无万元/亿元混用问题
- **异常值排查**：极端值已标注说明，非数据录入错误
